# Семинар 3. Нейросетевой отбор кандидатов

На этом семинаре мы поэкспериментируем с нейросетевым рекомендером, реализованным в rectools.

### План действий

1. Используем ноутбук `NeuralCandgenSeminar.ipynb`, для того, чтобы обучить рекомендеры
   1. Загружаем в Google Drive датасет, собранный с симулятора
   2. Загружаем в Google Drive данные треков из botify/data/tracks.json
   3. Обучаем модели Popular, LightFM и SASRec
   4. Сравниваем рекомендеры по офлайн метрикам и проверяем адекватность рекомендаций на примерах.
   5. Выгружаем top-10 соседей для каждого трека.
    
2. Реализуем новый рекомендер `I2IRecommender`
   1. Загружаем рекомендации в базу I2I рекомендаций в `Redis`
   2. Загружаем историю прослушивания пользователя в `Redis`
   3. При запросе ищем сэмплируем из истории трек пропорционально времени прослушивания
   4. Рекомендуем похожие на него треки, но такие, которые пользователь не слушал
    
3. Создаем A/B эксперимент.
    1. В `botify.experiment.Experiments` создаем эксперимент `I2I`.
    2. В `botify.server.NextTrack` используем этот эксперимент для выбора рекомендера.
       В этом эксперименте будут участвовать рекомендеры на основе lightfm и SASRec.
    3. Запускаем рекомендер и симулятор.
   
4. Используем ноутбук `Week1Seminar.ipynb` для анализа.
    1. Запускаем анализатор для эксперимента `I2I`.
