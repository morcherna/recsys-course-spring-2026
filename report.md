## Homework 2 Report

# Abstract

Предложено улучшение сервиса Botify с помощью двухуровневого ML-ансамбля на основе LightGBM (регрессор времени прослушивания + классификатор skip). Модель обучается на логах симулятора и ранжирует I2I-кандидатов, формируя улучшенные рекомендации. Рекомендер ContextualIndexedRecommender динамически выбирает треки из ML-улучшенных I2I с учётом истории пользователя и фильтрацией уже прослушанного. A/B эксперимент показал статистически значимое улучшение метрики mean_time_per_session на **+12.4%** по сравнению с SasRec-I2I.

# Детали

## Пайплайн обработки данных и обучения
1. Из логов симулятора извлекаются события прослушивания (3M+ записей) и формируются обучающие примеры с признаками на основе эмбеддингов треков.
2. Признаки включают:
   - **Эмбеддинг-фичи**: косинусные сходства истории с треком (3 варианта взвешивания — равномерное, exponential decay, веса по времени прослушивания), сходство с user embedding.
   - **Сессионные**: позиция в сессии, среднее/стд время, skip rate (recent и общий), тренд времени.
   - **Пользовательские**: среднее время, skip rate, уникальность треков.
   - **Популярность**: количество прослушиваний, среднее время, skip rate для трека.
3. Обучается ансамбль из двух LightGBM: регрессор для предсказания listen_time (RMSE=0.225) и классификатор skip (ROC-AUC=0.91). Итоговый score = predicted_time × (1 − skip_probability).
4. ML-модель ранжирует кандидатов SasRec-I2I, результат сохраняется в improved_i2i.jsonl и загружается в Redis.

## Рекомендер ContextualIndexedRecommender
- Загружает историю пользователя из Redis.
- Выбирает якоря: треки с listen_time > 0.3 (лайкнутые) с весами по времени; если таких нет — последние 5 треков.
- Случайно выбирает якорь (с весами) и достаёт из ML-улучшенных I2I топ-1 непрослушанный трек.
- Если не найден — перебирает до 5 якорей, затем fallback на случайный.
